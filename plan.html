Keepsake

Keepsake is an internal photo album, feed, and comment app: a private social space for posts, albums, uploaded photos, and conversation.

Architecture

Backend

  • Spring Boot with Spring Modulith
  • Maven, Java 21
  • PostgreSQL persistence
  • Liquibase database migrations
  • Session authentication with Spring Security

Frontend

  • React, TypeScript, Vite
  • Served by Spring Boot in production
  • Tailwind CSS and shadcn/ui components
  • Vite proxy for local API development

Modules

  • identity: users, roles, account management, seeded dev admin.
  • feed: text posts, post photo attachments, feed queries.
  • albums: albums, album metadata, album photo membership.
  • media: photo metadata, local filesystem storage, authenticated image serving.
  • comments: comments on posts and photos, editable with an edited marker.

V1 Product Rules

  • All authenticated users can view all posts, albums, photos, and comments.
  • Owners can edit or delete their own content; admins can edit or delete anything.
  • Feed contains intentional posts only. Album uploads do not automatically create feed entries.
  • Posts and comments are plain text with preserved line breaks.
  • Photos can belong to one parent in v1: either a post or an album.
  • Comments attach to posts and photos, not albums.
  • Deletes are soft deletes in the database. Local image files are retained in v1.
  • No reactions, notifications, private sharing, or threaded comments in v1.

Local Development

  • Docker Compose runs PostgreSQL for local development.
  • Tests use unit tests plus selected Testcontainers-backed integration tests.
  • Uploaded images are stored under a configurable local directory, defaulting to ./data/uploads.
